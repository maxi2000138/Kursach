# Быстрый старт

## Локальная разработка с Docker

1. **Клонируй/скопируй проект**

2. **Настрой переменные окружения** (опционально, можно в `docker-compose.yml`):
   - `SMTP_USERNAME` - email для отправки
   - `SMTP_PASSWORD` - пароль от email
   - `FROM_EMAIL` - отправитель

3. **Запусти контейнеры:**
```bash
docker-compose up -d
```

4. **Установи зависимости:**
```bash
docker-compose exec web composer install
```

5. **Открой в браузере:**
   - Приложение: http://localhost:8080
   - phpMyAdmin: http://localhost:8081

6. **База данных уже создана автоматически** из `sql/schema.sql`

## Тестирование

1. **Зарегистрируйся:**
   - Перейди на `/register`
   - Введи email и пароль
   - Проверь почту (код будет в консоли Docker или в письме)

2. **Подтверди email:**
   - Введи код из письма на странице `/verify-email`

3. **Войди в систему:**
   - Используй `/login`

4. **Восстанови пароль:**
   - `/forgot-password` → введи email
   - Проверь почту → введи код
   - Установи новый пароль

## Настройка SMTP для тестирования

Для локальной разработки можно использовать:
- **Gmail SMTP** (нужно включить "Ненадежные приложения")
- **Mailtrap** (для тестирования)
- **MailHog** (локальный SMTP сервер)

Или временно отключи отправку email и выводи код в консоль для тестирования.

## Стоп контейнеры

```bash
docker-compose down
```

## Очистка данных

```bash
docker-compose down -v
```

Это удалит все данные БД.
